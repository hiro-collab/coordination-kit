# Profile Promotion Templates

Use this folder after a run when you are deciding what should carry forward into the next run.

Do not start here for a first run. Most of the time, first write a candidate and stop there.

Read this folder in order:

1. `PROFILE_CANDIDATES_TEMPLATE.md`
2. `PROFILE_PROMOTION_REVIEW_TEMPLATE.md`
3. `TEAM_PROFILE_TEMPLATE.md`
4. `PLAYBOOK_TEMPLATE.md`

## What Each File Is For

- `PROFILE_CANDIDATES_TEMPLATE.md`: parking lot for possible carry-forward after a run
- `PROFILE_PROMOTION_REVIEW_TEMPLATE.md`: review gate before anything becomes durable
- `TEAM_PROFILE_TEMPLATE.md`: team-local overlay for repeated team behavior
- `PLAYBOOK_TEMPLATE.md`: reusable but conditional method

## Rules

- Do not promote directly from `WORKING_MEMORY`.
- Prefer a candidate first, then review.
- Keep the base profile thin.
- If the content is really a method, move it to a playbook instead of personality.
- If the content is useful for one team only, keep it in a team profile instead of the shared base.
