# Setup Templates

Use these files to bootstrap a run before lower-node execution starts.

Read this folder in order:

1. `MISSION_BRIEF_TEMPLATE.md`
2. `PUBLIC_TRIAL_BRIEF_TEMPLATE.md` if the run is comparative
3. `CONSTITUTION_TEMPLATE.md` when a stable shared guardrail already exists
4. `PERSONALITY_TEMPLATE.md` when a node needs durable style
5. `BELIEFS_TEMPLATE.md` when selected durable beliefs already exist
6. `SHARED_NOTE_TEMPLATE.md`
7. `INTEGRATION_TEMPLATE.md`
8. `WORKING_MEMORY_TEMPLATE.md`
9. `LOG_TEMPLATE.md`
10. `00_INDEX_TEMPLATE.md`

For a normal first run, start with `MISSION_BRIEF_TEMPLATE.md`, `SHARED_NOTE_TEMPLATE.md`, and `INTEGRATION_TEMPLATE.md`. Add durable profile files only when they already exist or the team truly needs them.

## What Each File Is For

- `MISSION_BRIEF_TEMPLATE.md`: generic mission brief for a normal run
- `PUBLIC_TRIAL_BRIEF_TEMPLATE.md`: comparative or public trial brief when matched public constraints are needed
- `CONSTITUTION_TEMPLATE.md`: hard guardrails that should rarely change
- `PERSONALITY_TEMPLATE.md`: durable style and decision posture separated from task-local memory
- `BELIEFS_TEMPLATE.md`: selected durable beliefs for base or team scope
- `SHARED_NOTE_TEMPLATE.md`: visible shared assumptions, owners, and cross-node boundaries
- `INTEGRATION_TEMPLATE.md`: integration status and coordination notes
- `WORKING_MEMORY_TEMPLATE.md`: per-node working memory file
- `LOG_TEMPLATE.md`: per-node activity log
- `00_INDEX_TEMPLATE.md`: directory-level navigation when files begin to sprawl

Start ordinary runs with `MISSION_BRIEF_TEMPLATE.md`, then `SHARED_NOTE_TEMPLATE.md`, then `INTEGRATION_TEMPLATE.md`.
Use `CONSTITUTION`, `PERSONALITY`, and `BELIEFS` only when you intentionally want carry-forward across runs.
