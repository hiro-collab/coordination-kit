# Setup Templates

Use these files to bootstrap a run before lower-node execution starts.

Read this folder in order:

1. `MISSION_BRIEF_TEMPLATE.md`
2. `PUBLIC_TRIAL_BRIEF_TEMPLATE.md` if the run is comparative
3. `PERSONALITY_TEMPLATE.md`
4. `SHARED_NOTE_TEMPLATE.md`
5. `INTEGRATION_TEMPLATE.md`
6. `WORKING_MEMORY_TEMPLATE.md`
7. `LOG_TEMPLATE.md`
8. `00_INDEX_TEMPLATE.md`

## What Each File Is For

- `MISSION_BRIEF_TEMPLATE.md`: generic mission brief for a normal run
- `PUBLIC_TRIAL_BRIEF_TEMPLATE.md`: comparative or public trial brief when matched public constraints are needed
- `PERSONALITY_TEMPLATE.md`: persistent behavior policy separated from task-local memory
- `SHARED_NOTE_TEMPLATE.md`: visible shared assumptions, owners, and cross-node boundaries
- `INTEGRATION_TEMPLATE.md`: integration status and coordination notes
- `WORKING_MEMORY_TEMPLATE.md`: per-node working memory file
- `LOG_TEMPLATE.md`: per-node activity log
- `00_INDEX_TEMPLATE.md`: directory-level navigation when files begin to sprawl
